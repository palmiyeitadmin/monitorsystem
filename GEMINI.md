# ERA Monitor System Rules

This document defines the strict coding and design standards for the ERA Monitor project. All future development must adhere to these rules to maintain the project's premium quality and architectural integrity.

## 1. Technology Stack & Architecture
- **Framework**: Next.js 15+ (App Router). Use server components by default; add `'use client'` only when interactivity is required.
- **Language**: TypeScript. Strict type checking is mandatory. No `any` types allowed.
- **Styling**: Tailwind CSS v4. Use `@import "tailwindcss";` in `globals.css`.
- **Icons**: Google Material Symbols Outlined (`material-symbols-outlined`).
- **State Management**: React Context or Zustand for global state (e.g., AuthContext).

## 2. Design Aesthetics & Philosophy
**CRITICAL**: The design must be "Premium", "Vibrant", and "Dynamic".
- **Visual Style**:
    - Use **Glassmorphism** (translucent backgrounds with blur) for cards and overlays.
    - **Gradients**: Use subtle, rich gradients (e.g., radial gradients for backgrounds) to add depth. Avoid flat, solid colors for large areas.
    - **Colors**: Use the defined OKLCH color palette in `globals.css`.
        - Primary: `#28aae2` (Cyan/Blue)
        - Dark Background: `#111c21`
        - Surface: `#1b2b32`
    - **Typography**: `Inter` font family. Clean, legible, and modern.
- **Animations**:
    - Implement micro-interactions (hover states, active states).
    - Use custom animations like `animate-float` and `animate-pulse-slow`.
    - Transitions should be smooth (`duration-200`, `duration-300`).
- **Prohibited**:
    - Generic/Bootstrap-like designs.
    - Plain, flat white/gray backgrounds without texture or depth.
    - Default browser fonts.

## 3. Component & Layout Standards
- **Directory Structure**:
    - Admin routes: `src/app/admin/...`
    - User routes: `src/app/user/...`
    - Shared components: `src/components/...`
- **Responsiveness**:
    - **Mobile-First**: Ensure all layouts work on mobile.
    - **Sidebar**:
        - Desktop: Collapsible sidebar (Expanded/Collapsed states).
        - Mobile/Tablet: Hidden sidebar with Hamburger menu + Bottom Navigation Bar.
    - **Navigation**:
        - Use `Link` from `next/link`.
        - Active states must be clearly visible (e.g., `bg-primary/20 text-primary`).

## 4. Coding Conventions
- **File Naming**: Kebab-case for directories and files (e.g., `user-profile`, `not-found.tsx`).
- **Component Naming**: PascalCase (e.g., `AdminLayout`, `StatusCard`).
- **Error Handling**:
    - Use custom `not-found.tsx` and `error.tsx` pages that match the premium design.
    - Do not use default Next.js error pages.
- **Imports**: Use absolute imports with `@/` alias (e.g., `@/components/ui/button`).

## 5. Specific Implementation Rules
- **Tailwind v4**: Do not use `@tailwind` directives. Use CSS variables for theme configuration in `globals.css`.
- **Placeholder Images**: Do NOT use external placeholder services (e.g., `via.placeholder.com`). Use local SVGs or CSS-based placeholders to prevent console errors and maintain privacy.
- **Forms**: All inputs must have proper focus states (`focus:ring-2 focus:ring-primary`).

## 6. Verification Checklist
Before marking a task as complete, verify:
1.  **Build**: Does `npm run build` pass without errors?
2.  **Lint**: Are there any ESLint or TypeScript errors?
3.  **Mobile**: Does the layout break on 375px width?
4.  **Console**: Are there any errors in the browser console?
5.  **Aesthetics**: Does it look "Premium"?
